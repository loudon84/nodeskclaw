# NoDeskClaw PRD v6.2.1 Hotfix

## Expert MCP 普通成员调用报错修复

版本：v6.2.1 Hotfix
项目：nodeskclaw
目标模块：Expert MCP Gateway / ExpertRunService / HermesTask 调度
优先级：P0
目标：修复普通组织成员调用 Expert MCP 时错误返回 `errors.hermes.agent_not_bound` 的问题。

---

# 1. 问题描述

copilot-desktop 使用普通成员账号调用：

```http
POST /api/v1/expert/mcp/{slug}
```

返回：

```text
("('任务只能下发给已绑定的 Hermes Agent AI 员工实例', 'errors.hermes.agent_not_bound')", 'errors.hermes.agent_not_bound')
```

该行为不符合 Expert MCP Gateway 设计。

Expert MCP Gateway 应允许真实员工、管理员、AI 员工等组织成员调用已发布 Expert / Skill。
调用者只负责发起任务，任务执行者应由 Expert 绑定的 Hermes Agent Runtime 决定。

---

# 2. 根因

当前 `ExpertRunService` 创建 HermesTask 时，直接使用：

```python
agent_id = expert.hermes_agent_id
```

或：

```python
agent_id = team.hermes_agent_id
```

但实际含义不一致：

```text
expert.hermes_agent_id / team.hermes_agent_id
  = HermesAgentInstance.id

HermesTask.agent_id
  = Instance.id
```

`HermesAgentAdapter.submit_run()` 和 `HermesBoundAgentScopeService.assert_bound_instance()` 需要的是 `Instance.id`。

因此普通成员调用时，Worker 使用错误的 `agent_id` 下发任务，导致：

```text
errors.hermes.agent_not_bound
```

---

# 3. 修复目标

## 3.1 正确调用模型

```text
当前登录用户 user_id
  用于权限、审计、调用日志

Expert.hermes_agent_id
  用于定位 Expert 绑定的 HermesAgentInstance

HermesAgentInstance.instance_id
  用于创建 HermesTask.agent_id

HermesTask.agent_id
  用于 Worker 下发到真实 Hermes Agent Runtime
```

## 3.2 修复后结果

普通成员调用 Expert MCP 时：

```text
1. 通过 expert_skill:invoke 权限校验。
2. 后端解析 Expert 绑定的 HermesAgentInstance。
3. 使用 HermesAgentInstance.instance_id 创建 HermesTask。
4. Worker 正常下发任务到对应 Hermes Agent。
5. 返回 taskId / eventSseUrl / artifactUrl。
```

# 5. 实现要求

## 5.1 ExpertRunService 新增执行 Agent 解析方法

在 `ExpertRunService` 中新增：

```python
async def _resolve_execution_agent(self, org_id: str, hermes_agent_id: str):
    ...
```

职责：

```text
1. 根据 org_id + hermes_agent_id 获取 HermesAgentInstance。
2. 校验 HermesAgentInstance 存在。
3. 校验 HermesAgentInstance.instance_id 存在。
4. 返回 HermesAgentInstance。
```

建议错误码：

```text
errors.expert.agent_instance_not_bound
```

错误信息：

```text
Expert 绑定的 Hermes Agent 尚未关联 AI 员工实例
```

---

## 5.2 修复 start_expert_skill_run

当前逻辑：

```python
agent_profile = await self.catalog.resolve_agent_profile(org_id, expert)
...
agent_id = expert.hermes_agent_id
```

修改为：

```python
agent = await self._resolve_execution_agent(org_id, expert.hermes_agent_id)
agent_profile = agent.profile_name

run_result = await self._create_task_run(
    org_id=org_id,
    user_id=user_id,
    skill=skill,
    agent_id=agent.instance_id,
    agent_profile=agent_profile,
    arguments=arguments,
    client_context=client_context,
    route_snapshot=route_snapshot,
    output_policy=output_policy,
    log=log,
)
```

要求：

```text
HermesTask.agent_id = agent.instance_id
HermesTask.profile_id = agent.profile_name
route_snapshot.hermes_agent_instance_id = expert.hermes_agent_id
route_snapshot.agent_profile = agent.profile_name
```

# 6. 权限要求

普通成员可以调用 Expert MCP，但必须满足：

```text
expert:view
expert_skill:view
expert_skill:invoke
```

普通成员不需要：

```text
hermes_agent:manage
hermes_agent:health_check
hermes_agent:drain
```

权限边界：

```text
user_id
  只用于权限、审计、调用日志

Expert 绑定 Runtime
  决定实际任务执行 Agent
```

# 8. 测试用例

## 8.1 普通成员调用单专家

前置：

```text
1. 用户 role = member。
2. Expert 已发布。
3. ExpertSkill public=true。
4. ExpertSkill call_enabled=true。
5. Expert 绑定 HermesAgentInstance。
6. HermesAgentInstance.instance_id 不为空。
```

断言：

```text

2. 返回 eventSseUrl。
3. 返回 artifactUrl。
4. HermesTask.user_id = 普通成员 user_id。
5. HermesTask.agent_id = HermesAgentInstance.instance_id。
6. routing_metadata.route_snapshot.hermes_agent_instance_id = Expert.hermes_agent_id。
```

## 8.2 普通成员调用专家团队

前置：

```text
1. 用户 role = member。

3. ExpertTeamSkill public=true。
4. ExpertTeamSkill call_enabled=true。
5. ExpertTeam 绑定 HermesAgentInstance。
6. HermesAgentInstance.instance_id 不为空。
```

断言：

```text

2. HermesTask.agent_id = HermesAgentInstance.instance_id。
3. routing_metadata.route_snapshot.catalog_kind = expert_team。
```

# 10. Cursor 实施指令

请修复 v6.2 Expert MCP Event Stream 调用中的 `errors.hermes.agent_not_bound` 问题。

根因是 `ExpertRunService` 创建 HermesTask 时把 `expert.hermes_agent_id` / `team.hermes_agent_id` 直接传给 `TaskService.create_task(agent_id=...)`。该字段是 `HermesAgentInstance.id`，但 `HermesTask.agent_id` 必须是 `Instance.id`。

修改要求：

```text
1. 在 ExpertRunService 中新增 _resolve_execution_agent(org_id, hermes_agent_id)。
2. 通过 HermesAgentInstance 获取 instance_id 和 profile_name。
3. start_expert_skill_run 使用 agent.instance_id 创建 HermesTask。
4. start_team_skill_run 使用 agent.instance_id 创建 HermesTask。
5. route_snapshot.hermes_agent_instance_id 继续保留原 hermes_agent_id。
6. route_snapshot.agent_profile 使用 agent.profile_name。
7. 不要要求当前 user 是 AI 员工。
8. 不要跳过 HermesBoundAgentScopeService 校验。
9. 增加普通 member 调用 Expert / ExpertTeam 的测试。
10. 增加未绑定 instance_id 的错误测试。
```

完成后，普通成员调用 Expert MCP 应返回 taskId/eventSseUrl/artifactUrl，并正常进入 HermesTask Worker 执行。
