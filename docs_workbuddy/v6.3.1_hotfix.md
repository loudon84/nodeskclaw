# NoDeskClaw 修复方案 PRD v6.3.1 Hotfix

版本：v6.3.1 hotfix
范围：`nodeskclaw-backend`
主题：修复 Desktop Runtime Skill 误入 `/v1/runs` 旧链路，并增强任务创建链路日志与前端原始上下文快照
实施目标：让 Cursor 可直接按本文完成后端修复

---

## 1. 背景

v6.3 的目标是：

```text
Desktop
  → NoDeskClaw MCP tools/call
  → Runtime Skill 固定路由 hermes_api_server
  → HermesTask async_event
  → Hermes API_SERVER /v1/chat/completions
  → NoDeskClaw Task SSE Timeline
  → Result / Artifacts
```

但当前异常日志显示：

```text
POST http://192.168.102.247:29204/v1/runs "HTTP/1.1 202 Accepted"
read_run_events stream error
get_run_status failed
```

说明 Desktop 发起的任务仍进入了旧的 `agent_run_stream` 路径。当前 Worker 逻辑是：只有 `route_snapshot.route_type == "hermes_api_server"` 才会进入 `_execute_api_server_task()`，否则会进入 `_execute_agent_run_stream()`。
而 `_execute_agent_run_stream()` 会调用 `adapter.submit_run()`，最终触发 Hermes Agent `/v1/runs`。

本次 hotfix 只修 NoDeskClaw 后端，不依赖 Desktop 先修好。后端要做到：即使 Desktop 误传或误走旧链路，也能保存上下文、明确失败、快速定位，而不是静默进入 `/v1/runs` 后超时。

---

## 2. 修复目标

### 2.1 必须达成

```text
1. Runtime Skill 创建的 HermesTask 必须写入明确执行契约。
2. execution_contract.runtime_invocation = chat_completions 时，Worker 禁止进入 /v1/runs。
3. hermes_tasks 必须保存前端传入的请求上下文快照，方便排查。
4. 任务创建、路由解析、Worker 执行分流必须有结构化日志。
5. tools/list 必须返回 Runtime Skill 固定路由元数据。
6. httpx.Timeout 半参数写法必须修复。
```

### 2.2 不做

```text
1. 不改 Desktop UI。
2. 不删除旧 expert_agent_event_stream。
3. 不禁止真正的专家团队 / 旧 Expert Gateway 兼容任务使用 /v1/runs。
4. 不保存 Authorization、Cookie、API Key、Token 明文。
5. 不把远端 workspace 文件直接暴露给 Desktop。
```

---

## 3. 核心判断规则

后端统一采用以下规则：

```text
只要 skill.source_type = hermes_api_server，
就必须走 Runtime Skill 固定路由。

正确：
  route_snapshot.route_type = hermes_api_server
  execution_contract.runtime_invocation = chat_completions
  timeline_provider = nodeskclaw_task_events

错误：
  Runtime Skill 进入 expert_agent_event_stream
  Runtime Skill 进入 /v1/runs
  Desktop arguments 中包含 _routing / _execution / route_config
```

### 4.2 request_snapshot 结构

`request_snapshot` 保存前端 / MCP 调用进入 NoDeskClaw 时的请求快照。

```json
{
  "trace_id": "req_...",
  "received_at": "2026-06-30T17:44:49Z",
  "entrypoint": "mcp_skill_gateway",
  "jsonrpc_method": "tools/call",
  "tool_name": "hermes_xieyi__customer-profiling",
  "client": {
    "source": "copilot-desktop",
    "client_name": "copilot-desktop",
    "client_version": "v7.5",
    "device_id": "desktop-001",
    "conversation_id": "local-session-001",
    "workspace_id": "default"
  },
  "auth": {
    "auth_type": "mcp_client_token",
    "token_id": "xxx",
    "token_prefix": "abcd****"
  },
  "arguments": {
    "prompt_preview": "分析研华科技的客户画像...",
    "prompt_sha256": "sha256:...",
    "context": {},
    "argument_keys": ["prompt", "context"],
    "forbidden_keys": []
  },
  "headers_safe": {
    "x-client-name": "copilot-desktop",
    "x-client-version": "v7.5"
  }
}
```

规则：

```text
1. prompt 可保存 preview + sha256。
2. context 可保存完整业务上下文，但默认最大 32KB。
3. 超过 32KB 时截断，并写入 truncated=true。
4. Authorization / Cookie / API Key / Token 明文不得入库。
5. 附件只保存 name / mime / size / sha256，不保存文件内容。
6. 原始 arguments 仍由 hermes_tasks.arguments 保存；request_snapshot 用于诊断快照。
```

### F4. Worker 增加执行契约保护

文件：

```text
app/services/hermes_skill/hermes_task_worker.py
```

在 `_execute_task()` 获取 `route_snapshot` 后增加保护：

```python
metadata = task.routing_metadata or {}
contract = metadata.get("execution_contract") or {}
route_type = route_snapshot.get("route_type")

if contract.get("runtime_invocation") == "chat_completions" and route_type != "hermes_api_server":
    await task_service.mark_failed(
        task,
        error_code="RUNTIME_ROUTE_CONTRACT_VIOLATION",
        error_message=(
            "Runtime Skill 执行契约要求 chat_completions，"
            f"但 route_snapshot.route_type={route_type or '<empty>'}，已阻止进入 /v1/runs"
        ),
    )
    await event_service.write_event(
        task_id=task.id,
        org_id=task.org_id,
        event_type="task.failed",
        payload={
            "error_code": "RUNTIME_ROUTE_CONTRACT_VIOLATION",
            "route_type": route_type,
            "expected_route_type": "hermes_api_server",
            "request_trace_id": task.request_trace_id,
        },
        source="worker",
    )
    await db.flush()
    return
```

目的：

```text
1. Runtime Skill 不再静默进入 /v1/runs。
2. 错误能立即显示在 SSE Timeline。
3. 日志能通过 request_trace_id 关联前端原始上下文。
```

### F6. httpx.Timeout Hotfix

文件：

```text
app/services/hermes_skill/hermes_agent_adapter.py
```

当前 `read_run_events()` 和 `get_run()` 的 `httpx.Timeout` 只传了 `connect/read`，会触发：

```text
httpx.Timeout must either include a default, or set all four parameters explicitly.
```

源码中 `read_run_events()` 确实只传了 `connect`、`read`。
`get_run()` 也是同类问题。

新增统一方法：

```python
def build_hermes_timeout(*, stream: bool = False) -> httpx.Timeout:
    read_timeout = (
        settings.HERMES_AGENT_DEFAULT_TIMEOUT_SECONDS
        if stream
        else settings.HERMES_AGENT_READ_TIMEOUT_SECONDS
    )
    return httpx.Timeout(
        connect=settings.HERMES_AGENT_CONNECT_TIMEOUT_SECONDS,
        read=read_timeout,
        write=settings.HERMES_AGENT_READ_TIMEOUT_SECONDS,
        pool=settings.HERMES_AGENT_READ_TIMEOUT_SECONDS,
    )
```

替换：

```text
submit_run()
cancel_run()
read_run_events()
get_run()
```

说明：

```text
这是旧 /v1/runs 兼容路径的稳定性修复。
它不能替代 Runtime Skill 固定路由修复。
```

## 10. 完成定义

v6.3.1 hotfix 完成后，任何 Desktop Runtime Skill 调用都必须满足：

```text
HermesTask.request_trace_id 非空
HermesTask.request_snapshot 可查
HermesTask.route_diagnostics 可查
routing_metadata.route_snapshot.route_type = hermes_api_server
routing_metadata.execution_contract.runtime_invocation = chat_completions
Worker 调用 /v1/chat/completions
Desktop 订阅 NoDeskClaw /hermes/tasks/{task_id}/events
```

如果以上任一条件不满足，任务必须快速失败并输出可诊断日志，不允许继续进入 `/v1/runs` 后等待 timeout。
