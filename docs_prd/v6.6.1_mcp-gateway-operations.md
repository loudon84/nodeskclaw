# PRD 2：nodeskclaw v6.6.1_mcp-gateway-operations

## 1. 版本信息

版本号：v6.6.1_mcp-gateway-operations
模块范围：nodeskclaw backend
主题：MCP Gateway 运营化与安全治理
依赖版本：v6.6 MCP Skill Gateway E2E 已完成并测试成功
目标对象：nodeskclaw、Copilot Desktop、本地 Hermes Agent、Docker Hermes 实例

---

## 2. 背景

v6.6 已完成 nodeskclaw 对外暴露 MCP Skill Gateway 的基础链路：

1. `/api/v1/system/info` 可被 Copilot Desktop 识别。
2. `/api/v1/hermes/mcp` 可被 Desktop Proxy 调用。
3. MCP 支持基础 `initialize`、`tools/list`、`tools/call`。
4. 已能暴露 Hermes Docker 实例状态与 skills 清单。
5. Copilot Desktop 可以通过 MCP Proxy 调用 nodeskclaw tools。

当前需要从“可调用”升级为“可运营、安全可控”。

主要缺口：

1. tools/list 缺少 permission、riskLevel、category 等治理字段。
2. tools/call 缺少统一错误码。
3. 工具权限分层需要稳定。
4. instance_ref 解析规则需要标准化。
5. MCP 调用需要审计。
6. 写操作工具需要被明确隔离。
7. GeneHub 注册能力需要预留，但不能混入本阶段落盘实现。

---

## 3. 产品目标

v6.6.1 目标是让 nodeskclaw MCP Skill Gateway 具备稳定的企业级调用治理能力。

### 3.1 核心目标

1. 标准化 MCP Gateway tools/list 输出。
2. 增加 tool permission 与 riskLevel。
3. 增加工具调用权限控制。
4. 增加 MCP 调用审计。
5. 增加统一错误码。
6. 稳定 instance_ref 解析。
7. 支持 Copilot Desktop 一键诊断。
8. 为 v6.8 GeneHub Skill Registration 预留接口与工具命名。

---

## 4. 非目标

本版本不做：

1. 不实现 GeneHub skill 安装到用户本地 Hermes。
2. 不直接写用户电脑 `~/.hermes/skills`。
3. 不开放静默写操作工具。
4. 不做 skill 审核流。
5. 不做 skill marketplace UI。
6. 不做多租户复杂策略中心。
7. 不让 Hermes Agent 直接绕过 Copilot Desktop Proxy 调用内部管理接口。

---

## 5. 用户角色

| 角色              | 诉求                                   |
| --------------- | ------------------------------------ |
| Copilot Desktop | 通过 MCP Gateway 获取 tools、执行只读调用、展示诊断  |
| Hermes Agent    | 通过 Desktop Proxy 调用 nodeskclaw tools |
| IT 管理员          | 管理 Hermes Docker 实例、查看审计             |
| 运维人员            | 追踪 MCP 调用错误与耗时                       |
| 业务用户            | 通过 Hermes Agent 间接使用工具能力             |

## 7. 功能需求

## 7.1 System Info MCP Descriptor

接口：

```http
GET /api/v1/system/info
```

返回必须包含：

```json
{
  "service": "nodeskclaw",
  "mcp": {
    "endpoint": "/api/v1/hermes/mcp",
    "transport": "streamable_http",
    "protocolVersion": "2025-06-18",
    "healthEndpoint": "/api/v1/mcp/health"
  }
}
```

验收：

1. 未登录也可返回非敏感 descriptor。
2. descriptor 不包含 token。
3. endpoint 不写死主机，只返回路径。
4. Copilot Desktop 可据此拼接 remoteMcpUrl。

## 7.4 tools/list 标准化

tools/list 返回中每个 tool 增加治理字段。

结构：

```json
{
  "name": "hermes.skills.list",
  "description": "List installed skills of a bound Hermes instance",
  "inputSchema": {
    "type": "object",
    "properties": {
      "instance_ref": {
        "type": "string"
      }
    },
    "required": ["instance_ref"]
  },
  "annotations": {
    "category": "hermes",
    "permission": "read",
    "riskLevel": "low",
    "requiresApproval": false,
    "enabled": true
  }
}
```

第一批 tools：

```text
hermes.instances.list
hermes.instance.status
hermes.skills.list
```

预留但默认不开放：

```text
hermes.skills.install_builtin
hermes.skills.install_git
hermes.skills.install_zip
hermes.skills.uninstall
hermes.instance.restart
genehub.skills.search
genehub.skill.detail
genehub.skill.register_to_hermes
genehub.registration.status
```

## 7.7 instance_ref 解析规则

统一解析顺序：

```text
1. instance_id 精确匹配
2. container_name 精确匹配
3. profile 精确匹配
4. alias 精确匹配
5. 当前用户默认 Hermes 实例
```

如果匹配多个：

```text
返回 HERMES_INSTANCE_AMBIGUOUS
```

如果不存在：

```text
返回 HERMES_INSTANCE_NOT_FOUND
```

如果用户无权限：

```text
返回 HERMES_INSTANCE_FORBIDDEN
```

---

## 7.8 权限模型

工具权限分层：

```text
read:
  hermes.instances.list
  hermes.instance.status
  hermes.skills.list

write:
  hermes.skills.install_builtin
  hermes.skills.install_git
  hermes.skills.install_zip
  hermes.skills.uninstall

admin:
  hermes.instance.restart
  hermes.instance.rebind
```

v6.6.1 执行规则：

1. read tools 可执行。
2. write tools 在 tools/list 中可选择隐藏或标记 disabled。
3. admin tools 默认不返回。
4. write/admin 需要 v6.9 审批流后开放。
5. 所有 tools/call 必须校验用户组织、角色、实例权限。

## 7.9 MCP 调用审计

新增表：

```sql
CREATE TABLE mcp_call_logs (
  id UUID PRIMARY KEY,
  org_id UUID,
  user_id UUID,
  client_name TEXT,
  tool_name TEXT,
  permission TEXT,
  risk_level TEXT,
  instance_id TEXT,
  status TEXT NOT NULL,
  duration_ms INTEGER,
  input_summary JSONB,
  result_summary JSONB,
  error_code TEXT,
  error_message TEXT,
  created_at TIMESTAMPTZ NOT NULL
);
```

审计规则：

1. 每次 tools/call 必须写审计。
2. initialize 可选写审计。
3. tools/list 可按配置写审计。
4. 不记录 Authorization header。
5. 不记录 accessToken。
6. 不记录 WebUI 密码。
7. input_summary 需要脱敏。

## 7.12 GeneHub 注册能力预留

v6.6.1 只预留，不实现完整注册落盘。

预留 tools：

```text
genehub.skills.search
genehub.skill.detail
genehub.skill.register_to_hermes
genehub.registration.status
```

其中 `genehub.skill.register_to_hermes` 的未来行为：

```text
1. 检查 GeneHub skill 是否 approved。
2. 创建 hermes_skill_registration job。
3. 返回 registration_id。
4. 等待 Copilot Desktop 拉取并本地安装。
5. 不直接写用户本地 ~/.hermes/skills。
```

## 8. 安全要求

1. 所有 MCP 请求必须校验 accessToken。
2. 用户必须属于当前 org。
3. 用户必须有实例访问权限。
4. read tools 可直接调用。
5. write/admin tools 默认不可调用。
6. 所有 tools/call 必须写审计。
7. 不记录 Authorization header。
8. 不记录 accessToken。
9. 不记录 WebUI 访问密码。
10. 所有异常转成 JSON-RPC error。
11. 不返回 Python traceback 给客户端。

## 11. 验收标准

1. `/api/v1/system/info` 返回 MCP descriptor。
2. `/api/v1/mcp/health` 返回 tool count。
3. `/api/v1/hermes/mcp initialize` 成功。
4. `/api/v1/hermes/mcp tools/list` 返回至少 3 个 read tools。
5. tools/list 每个 tool 包含 category、permission、riskLevel、requiresApproval、enabled。
6. `hermes.instances.list` 可返回绑定 Hermes Docker 实例。
7. `hermes.instance.status` 可返回 zhang-zhen 实例状态。
8. `hermes.skills.list` 可返回 zhang-zhen skills 清单。
9. 未登录返回 JSON-RPC auth error。
10. 无实例权限返回 forbidden error。
11. 实例不存在返回 `HERMES_INSTANCE_NOT_FOUND`。
12. 每次 tools/call 写入 `mcp_call_logs`。
13. 审计日志不包含 token、密码、Authorization header。
14. write/admin tools 默认不可调用。
15. Copilot Desktop 一键诊断可成功读取 tools/list 与 invoke read tool。
