# PRD 2：nodeskclaw v6.6.2_genehub-mcp-registration

## 1. 版本信息

版本号：v6.6.2_genehub-mcp-registration
项目：nodeskclaw
模块范围：

```text
nodeskclaw-backend/app/api/mcp_skill_gateway/
nodeskclaw-backend/app/services/mcp_skill_gateway/
nodeskclaw-backend/app/api/desktop_genehub.py
nodeskclaw-backend/app/services/genehub_service.py
nodeskclaw-backend/app/services/hermes_desktop_sync_service.py
nodeskclaw-backend/app/schemas/genehub.py
```

依赖版本：

```text
v6.6.0：MCP Gateway E2E
v6.6.1：MCP Gateway operations
Desktop GeneHub Sync 基础能力
```

本版本目标：

```text
开放 GeneHub MCP tools，使 Hermes Agent 能通过 MCP Gateway 搜索 GeneHub skill、
查看 skill 详情、创建注册到 Hermes 的 install job，并查询安装状态。
```

---

## 2. 背景

当前 nodeskclaw 已完成：

```text
1. MCP Gateway descriptor。
2. /api/v1/hermes/mcp JSON-RPC。
3. initialize。
4. tools/list。
5. tools/call。
6. Hermes Docker read tools。
7. MCP tool registry。
8. MCP audit。
9. Desktop GeneHub API。
10. Desktop install job 创建、pending、claim、bundle、status、installed sync。
```

当前 GeneHub MCP tools 状态：

```text
genehub.skills.search              已登记，未启用
genehub.skill.detail               已登记，未启用
genehub.skill.register_to_hermes   已登记，未启用
genehub.registration.status        已登记，未启用
```

v6.6.2 的核心是把这些 tools 从 registry 预留变成可执行工具。

---

## 3. 产品目标

### 3.1 核心目标

```text
1. MCP tools/list 返回 GeneHub read tools。
2. Hermes Agent 可调用 genehub.skills.search。
3. Hermes Agent 可调用 genehub.skill.detail。
4. Hermes Agent 可调用 genehub.skill.register_to_hermes 创建 install job。
5. Hermes Agent 可调用 genehub.registration.status 查询 job 状态。
6. 所有 GeneHub MCP tool 调用写入 mcp_call_logs。
7. register_to_hermes 不直接安装，只创建 job。
8. Desktop 负责本地安装。
```

### 3.2 安全目标

```text
1. nodeskclaw 不直接写用户本地 ~/.hermes。
2. register_to_hermes 只创建 pending job。
3. 用户必须有组织权限与 profile 权限。
4. skill 必须对当前用户可见。
5. install job 必须绑定 desktop profile。
6. 所有工具调用都有审计。
```

---

## 4. 非目标

本版本不做：

```text
1. 不直接安装 skill 到用户本地。
2. 不做 Git/ZIP 任意安装。
3. 不开放 Hermes skills install_git / install_zip。
4. 不开放 Hermes instance restart。
5. 不做完整审批流。
6. 不做 GeneHub 发布审核后台。
7. 不做 Marketplace 前端页面。
```

```text

```

## 6. 功能需求

## 6.1 genehub.skills.search

### 用途

搜索当前用户可见、可安装到指定 Hermes profile 的 GeneHub skills。

### 输入

```json
{
  "query": "order",
  "profile_id": "optional",
  "category": "business",
  "tag": "crm"
}
```

字段说明：

```text
query：关键词，可选
profile_id：目标 Desktop Hermes profile，可选；为空时使用当前用户默认 active profile
category：分类，可选
tag：标签，可选
```

### 输出

```json
{
  "skills": [
    {
      "gene_slug": "contact-to-order",
      "gene_version": "1.0.0",
      "skill_name": "contact-to-order",
      "display_name": "Contact To Order",
      "description": "Convert customer contact text to structured order fields.",
      "category": "business",
      "tags": ["crm", "order"],
      "installed": false,
      "installed_version": null,
      "update_available": false,
      "permissions": {
        "can_install": true,
        "can_update": false,
        "can_uninstall": false
      }
    }
  ]
}
```

### 底层服务

复用：

```text
genehub_service.list_desktop_visible_skills()
```

### 验收

```text
1. 当前用户只能看到授权 skills。
2. profile_id 不存在返回错误。
3. 无权限 profile 返回 forbidden。
4. 返回结果不包含 bundle 文件内容。
5. 写入 MCP audit。
```

## 6.3 genehub.skill.register_to_hermes

### 用途

由 Hermes Agent 发起 GeneHub skill 注册请求，nodeskclaw 创建 install job，等待 Copilot Desktop 拉取并本地安装。

### 输入

```json
{
  "gene_slug": "contact-to-order",
  "profile_id": "default",
  "version": "latest",
  "action": "install"
}
```

字段说明：

```text
gene_slug：必填
profile_id：目标 Desktop Hermes profile，可选；为空时使用默认 active profile
version：默认 latest
action：install | update | uninstall，本版本默认只允许 install/update
```

### 输出

```json
{
  "job_id": "job_xxx",
  "status": "pending",
  "gene_slug": "contact-to-order",
  "gene_version": "1.0.0",
  "skill_name": "contact-to-order",
  "profile_id": "default",
  "action": "install",
  "message": "Install job created. Copilot Desktop will apply it locally after user confirmation."
}
```

### 行为

```text
1. 校验 accessToken。
2. 校验当前 org。
3. 校验 profile 权限。
4. 校验 gene_slug 可见。
5. 校验用户 canInstall/canUpdate。
6. 创建 install job。
7. job source 标记为 mcp_agent_request。
8. 返回 job_id。
9. 不直接安装。
10. 写入 MCP audit。
```

### 幂等规则

如果相同用户、相同 profile、相同 gene_slug 存在未完成 job：

```text
pending / claimed / downloading / validating / installing
```

则不重复创建，直接返回已有 job。

如果已有 installed 且版本一致：

```text
返回 status=installed，不创建新 job。
```

如果已有 installed 但版本较旧：

```text
创建 update job。
```

## 16. Cursor 执行提示词

你现在执行 nodeskclaw v6.6.2_genehub-mcp-registration。

目标：
开放 GeneHub MCP tools，让 Hermes Agent 可以通过 nodeskclaw MCP Gateway 搜索 GeneHub skills、查看 skill 详情、创建注册到 Hermes 的 install job，并查询 install job 状态。nodeskclaw 只创建 job，不直接写用户本地 Hermes 文件；本地安装由 Copilot Desktop 完成。

强制约束：

1. nodeskclaw 不直接写 ~/.hermes。
2. register_to_hermes 只创建 install job。
3. 所有 GeneHub MCP tool 必须鉴权。
4. 所有 GeneHub MCP tool 必须写 audit。

6. 默认只开放 GeneHub read tools 和受控 register_to_hermes。
7. 不开放 hermes.skills.install_git / install_zip / uninstall。
8. 不返回 bundle 内容给 MCP detail。
9. 不记录 token、password、bundle content、script content。
10. Desktop 安装完成后通过现有 status API 回写 job 状态。
