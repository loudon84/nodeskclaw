## 3. 产品目标

v6.7 目标是让 MCP Gateway 支持写操作工具，但所有写操作与高风险操作必须先在 nodeskclaw 服务端授权。

核心目标：

```text
1. nodeskclaw 提供 MCP 写操作工具授权中心。
2. 管理员可审批用户 / Desktop / Profile / 实例 / Tool 的使用权限。
3. 审批通过后，Hermes Agent 可直接调用对应 MCP write tool。
4. copilot-desktop 不再做二次确认。
5. nodeskclaw 可撤销授权。
6. 授权撤销后，Hermes Agent 后续调用立即失败。
7. 所有授权、调用、撤销动作写入审计。
```

---

本版本不做：

```text
1. 不做 copilot-desktop 本地审批弹窗。
2. 不做 Desktop 二次确认。
3. 不让 nodeskclaw 写用户本地 ~/.hermes。
4. 不开放 hermes.instance.rebind。
5. 不开放任意无校验 install_zip / install_git 静默执行。
6. 不做多级审批流。
7. 不做消息通知系统。
8. 不做审批流 BPM 集成。
```

## 5. 审批模型

### 5.1 审批原则

```text
read tool：
  默认可用，不需要审批。

write tool：
  需要 nodeskclaw 服务端授权。

admin tool：
  需要 nodeskclaw 服务端授权，默认更严格。

copilot-desktop：
  只负责注册 MCP Gateway、注入 token、传递 Desktop/Profile 上下文。
  不负责审批。
```

### 5.2 授权粒度

授权必须支持以下维度：

```text
org_id
user_id
desktop_device_id
profile_id / profile_name
instance_id
tool_name
permission
risk_level
grant_status
expires_at
```

推荐授权匹配优先级：

```text
1. user_id + desktop_device_id + profile_id + instance_id + tool_name
2. user_id + profile_id + instance_id + tool_name
3. user_id + instance_id + tool_name
4. org_id + role + instance_id + tool_name
```

v6.7 P0 建议先实现：

```text
user_id + instance_id + tool_name
```

v6.7 P1 再补：

```text
desktop_device_id + profile_id / profile_name
```

```json

    "properties": {
```

## 9. tools/call 行为

## 9.1 未授权调用 write/admin tool

当 Hermes Agent 调用未授权写工具：

```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "method": "tools/call",
  "params": {
    "name": "hermes.skills.uninstall",
    "arguments": {
      "instance_ref": "zhang-zhen",
      "skill_name": "demo-skill"
    }
  }
}
```

nodeskclaw 行为：

```text
1. 校验用户身份。
2. 校验组织。
3. 校验实例权限。
4. 识别 tool 为 write/admin。
5. 查询 active grant。
6. 若无 active grant，创建 approval request。
7. 返回 MCP_TOOL_APPROVAL_REQUIRED。
8. 写入 mcp_call_logs。
9. 写入 mcp_tool_policy_events。
```

返回：

```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "error": {
    "code": -32023,
    "message": "Tool approval required",
    "data": {
      "errorCode": "MCP_TOOL_APPROVAL_REQUIRED",
      "approvalRequestId": "apr_xxx",
      "toolName": "hermes.skills.uninstall",
      "approvalMode": "server",
      "message": "Approval request created in nodeskclaw."
    }
  }
}
```

---

## 9.2 已授权调用 write/admin tool

若存在 active grant：

```text
1. 校验 grant 是否 active。
2. 校验 grant 未过期。
3. 校验 instance_id 与 tool_name 匹配。
4. 执行工具。
5. 写入 mcp_call_logs。
6. 写入 policy event grant_used。
7. 返回执行结果。
```

---

## 9.3 撤销授权后的调用

若 grant 已撤销：

```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "error": {
    "code": -32024,
    "message": "Tool grant revoked",
    "data": {
      "errorCode": "MCP_TOOL_GRANT_REVOKED",
      "toolName": "hermes.skills.uninstall",
      "revokedAt": "2026-xx-xxT00:00:00Z"
    }
  }
}
```

## 10. 写操作工具设计

## 10.1 hermes.skills.install_builtin

输入：

```json
{
  "instance_ref": "zhang-zhen",
  "skill_slug": "writer-basic"
}
```

行为：

```text
1. 校验 active grant。
2. 校验 skill_slug 属于 nodeskclaw 内置白名单。
3. 调用 Hermes external skill service 安装。
4. 返回安装结果。
```

输出：

```json
{
  "installed": true,
  "skill_slug": "writer-basic",
  "instance_id": "inst_xxx"
}
```

---

## 10.2 hermes.skills.uninstall

输入：

```json
{
  "instance_ref": "zhang-zhen",
  "skill_name": "demo-skill"
}
```

行为：

```text
1. 校验 active grant。
2. 校验 skill 存在。
3. 禁止卸载 protected skill。
4. 执行卸载。
5. 返回卸载结果。
```

protected skill 示例：

```text
hermes-agent
mcp-skill-gateway
genehub-runtime
```

---

## 10.3 hermes.instance.restart

输入：

```json
{
  "instance_ref": "zhang-zhen"
}
```

行为：

```text
1. 校验 active grant。
2. 校验实例属于 external_docker。
3. 校验实例生命周期模式允许 restart。
4. 执行 restart。
5. 返回 restart 状态。
```

输出：

```json
{
  "restarted": true,
  "instance_id": "inst_xxx",
  "status": "running"
}
```

## 11.3 审批通过

```http
POST /api/v1/mcp/tool-approval-requests/{request_id}/approve
```

请求：

```json
{
  "expires_at": null,
  "decision_comment": "允许该用户在 zhang-zhen 实例卸载非保护 skill",
  "constraints": {
    "allowed_skill_prefixes": [],
    "protected_skills": ["hermes-agent", "mcp-skill-gateway"]
  }
}
```

行为：

```text
1. 校验审批人 org admin/operator 权限。
2. 更新 request status=approved。
3. 创建 active grant。
4. 写 policy event。
5. 返回 grant。
```

---

## 11.4 审批拒绝

```http
POST /api/v1/mcp/tool-approval-requests/{request_id}/reject
```

## 11.6 撤销授权

```http
POST /api/v1/mcp/tool-grants/{grant_id}/revoke
```

请求：

```json
{
  "reason": "该用户不再需要卸载 skill 权限"
}
```

行为：

```text
1. grant_status=revoked。
2. 写 revoked_by / revoked_at / revoke_reason。
3. 写 policy event。
4. 后续 MCP tools/call 立即拒绝。
```

## 13. 权限要求

### 13.1 审批权限

```text
org admin：
  可审批所有 tool grants。

org operator：
  可审批 write tools。
  不可审批 admin tools。

member：
  不可审批。
```

### 13.2 Tool 执行权限

```text
1. 用户必须登录。
2. 用户必须属于当前 org。
3. 用户必须具备实例 viewer 权限。
4. write/admin tool 必须具备 active grant。
5. grant 必须未过期。
6. grant 必须未撤销。
7. tool constraints 必须满足。
```

## 17. 验收标准

```text

10. write/admin tools 不存在 active grant 时不得执行。
```
