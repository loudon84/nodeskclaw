项目：nodeskclaw

目标仓库：loudon84/nodeskclaw

模块范围：

## 2. 背景

v6.7 已完成：

```text
1. MCP Gateway descriptor。
2. /api/v1/hermes/mcp JSON-RPC。
3. tools/list。
4. tools/call。
5. MCP tool registry。
6. MCP audit。
7. GeneHub MCP tools。
8. Desktop GeneHub API。
9. install job 创建、pending、claim、bundle、status、installed sync。
```

当前阻断点：

```text
1. mcp_tool_registry.py 缺少 is_genehub_registry_tool，导致 GeneHubMcpToolProvider 导入存在风险。
2. genehub.skill.register_to_hermes 被通用 approval_service 按 server approval 拦截，与 Desktop 本地确认安装冲突。
3. pending jobs 返回字段不足，Desktop 无法稳定识别 source=mcp_agent_request。
4. bundle 下载接口要求 claimed 后访问，无法支持“确认前预览”。
5. ignore / cancel 状态需要服务端标准化。
6. installed skills sync 需要明确 server profile id 语义。
7. signature / bundle preview / audit 字段需要补齐。
```

---

## 3. 产品目标

### 3.1 核心目标

```text
1. 修复 GeneHub MCP Provider 导入缺陷。
2. genehub.skill.register_to_hermes 采用 desktop confirmation mode。
3. register_to_hermes 可直接创建 source=mcp_agent_request 的 pending install job。
4. Desktop pending jobs API 返回完整字段。
5. 新增 bundle-preview API，支持确认前预览。
6. ignore API 可将 pending job 标记为 cancelled。
7. installed skills sync 严格使用 server profile id。
8. MCP audit 与 install job audit 字段完整。
9. registration.status 可稳定返回 installed / failed / cancelled。
```

### 3.2 安全目标

```text
1. nodeskclaw 不直接写用户本地 ~/.hermes。
2. register_to_hermes 只创建 install job，不直接安装。
3. Desktop 负责本地安装与用户确认。
4. MCP detail 与 bundle-preview 不返回 bundle 正文。
5. 不记录 token、password、bundle content、script content。
6. 用户必须有 org、profile、skill 权限。
7. install_git / install_zip 继续禁用。
```

## 5. 核心链路

```text
Hermes Agent
  -> /api/v1/hermes/mcp tools/call
  -> genehub.skill.register_to_hermes
  -> GeneHubMcpToolProvider
  -> genehub_service.create_mcp_registration_job
  -> source=mcp_agent_request pending install job
  -> Desktop pending jobs API
```

## 6.3 genehub.skill.register_to_hermes 修复

行为要求：

```text
1. 校验 accessToken。
2. 校验当前 org。
3. 校验 profile 权限。
4. 校验 gene_slug 可见。
5. 校验 canInstall / canUpdate。
6. 创建 install job。
7. job source=mcp_agent_request。
8. job status=pending。
9. job 绑定 server profile id。
10. 返回 job_id。
11. 不直接安装。
12. 写入 MCP audit。
```

输入：

```json
{
  "gene_slug": "contact-to-order",
  "profile_id": "default",
  "version": "latest",
  "action": "install"
}
```

输出：

```json
{
  "job_id": "job_xxx",
  "status": "pending",
  "source": "mcp_agent_request",
  "gene_slug": "contact-to-order",
  "gene_version": "1.0.0",
  "skill_name": "contact-to-order",
  "profile_id": "profile_server_xxx",
  "profile_name": "default",
  "action": "install",
  "desktop_confirmation_required": true,
  "message": "Install job created. Copilot Desktop will apply it locally after user confirmation."
}
```

幂等规则：

```text
1. 相同 user + org + profile + gene_slug 存在 active job，返回已有 job。
2. active job 状态包括 pending / claimed / downloading / validating / installing。
3. 已 installed 且版本一致，返回 installed，不创建新 job。
4. 已 installed 但版本旧，创建 update job。
```

## 6.8 Ignore / Cancel API

接口：

```text
POST /api/v1/desktop/hermes/install-jobs/{job_id}/ignore
```

行为：

```text
1. 仅允许 pending 状态 ignore。
2. ignore 后 job status=cancelled。
3. 写入 client_report。
4. 写入 audit / status timeline。
5. registration.status 查询返回 cancelled。
```

请求：

```json
{
  "reason": "user_ignored",
  "client_report": {
    "desktopDeviceId": "desktop_xxx",
    "profileName": "default",
    "step": "user_ignored"
  }
}
```

返回：

```json
{
  "job_id": "job_xxx",
  "status": "cancelled",
  "error_code": null,
  "error_message": null
}
```

限制：

```text
1. claimed / installing / installed 不允许 ignore。
2. failed 不允许 ignore。
3. 无权限返回 forbidden。
```

修复 v6.7 GeneHub MCP Registration 的 E2E 阻断问题。nodeskclaw 必须让 genehub.skill.register_to_hermes 在 approvalMode=desktop 下直接创建 source=mcp_agent_request 的 pending install job，不触发 server approval 阻断；Desktop 负责本地用户确认。pending jobs API 必须返回 source/profileName/time/error 等完整字段。新增 bundle-preview API，允许 Desktop 在 pending 状态下预览 bundle metadata，但不 claim job、不返回正文内容。ignore API 必须把 pending job 标记为 cancelled。installed-skills sync 必须使用 server profile id。

强制约束：

1. nodeskclaw 不直接写 ~/.hermes。
2. register_to_hermes 只创建 install job。
3. approvalMode=desktop 不走 server approval request。
4. bundle-preview 不返回 bundle 正文。
5. bundle-preview 不 claim job。
6. pending jobs 必须返回 source=mcp_agent_request。
7. install_git / install_zip / uninstall 继续禁用。
8. 所有 GeneHub MCP tool 必须鉴权。
9. 所有 GeneHub MCP tool 必须写 audit。
10. 不记录 token、password、bundle content、script content。
