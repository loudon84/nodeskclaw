# 子 PRD A：nodeskclaw-backend

## v6.4.1_hotfix_mcp-backend

## 1. 目标

修复 backend 作为 MCP Skill Gateway 时，Desktop 通过 Local MCP Proxy 执行 `tools/list` 失败的问题。

backend 只负责：

```text
1. 提供 MCP Gateway descriptor
2. 提供认证后的 MCP JSON-RPC endpoint
3. 支持 initialize
4. 支持 tools/list
5. 根据用户权限过滤 tools
6. 返回结构化错误码
```

不处理 Desktop UI、Local MCP Proxy、Electron IPC。

---

## 2. 现状问题

Desktop 已能访问：

```text
GET /api/v1/health
GET /api/v1/system/info
```

但通过 Local MCP Proxy 调用：

```text
POST /mcp method=tools/list
```

返回：

```json
{
  "jsonrpc": "2.0",
  "id": "test",
  "error": {
    "code": -32012,
    "message": "MCP gateway request failed"
  }
}
```

backend 侧需要明确返回认证、权限、协议、tools/list 的具体错误，不能只返回 generic gateway failed。

---

## 3. 功能需求

### FR-BE-001：system/info 返回 MCP descriptor

接口：

```text
GET /api/v1/system/info
```

返回增加：

```json
{
  "success": true,
  "data": {
    "mcp": {
      "enabled": true,
      "name": "Coding MCP Gateway",
      "transport": "streamable_http",
      "endpoint": "/api/v1/mcp",
      "healthEndpoint": "/api/v1/mcp/health",
      "requiresAuth": true,
      "protocolVersion": "2025-06-18"
    }
  }
}
```

验收：

```text
1. endpoint 不为空。
2. transport 固定为 streamable_http。
3. endpoint 使用相对路径。
4. Desktop 可根据 backendBaseUrl + endpoint 合成 MCP URL。
```

---

### FR-BE-002：MCP endpoint 支持 Authorization

接口：

```text
POST /api/v1/mcp
```

请求头：

```text
Authorization: Bearer <access_token>
Content-Type: application/json
Accept: application/json, text/event-stream
```

无 token 或 token 无效返回：

```json
{
  "jsonrpc": "2.0",
  "id": "xxx",
  "error": {
    "code": -32001,
    "message": "MCP_UNAUTHORIZED",
    "data": {
      "errorCode": "MCP_UNAUTHORIZED",
      "reason": "Missing or invalid Authorization header"
    }
  }
}
```

验收：

```text
1. 无 Authorization 必须返回 MCP_UNAUTHORIZED。
2. 无效 token 必须返回 MCP_UNAUTHORIZED。
3. 不能返回 generic MCP gateway request failed。
```

---

### FR-BE-003：支持 initialize

请求：

```json
{
  "jsonrpc": "2.0",
  "id": "init-1",
  "method": "initialize",
  "params": {
    "protocolVersion": "2025-06-18",
    "capabilities": {},
    "clientInfo": {
      "name": "smc-copilot-desktop",
      "version": "v6.4.1"
    }
  }
}
```

响应：

```json
{
  "jsonrpc": "2.0",
  "id": "init-1",
  "result": {
    "protocolVersion": "2025-06-18",
    "capabilities": {
      "tools": {
        "listChanged": true
      }
    },
    "serverInfo": {
      "name": "nodeskclaw-mcp-skill-gateway",
      "version": "v6.4.1"
    }
  }
}
```

验收：

```text
1. 带有效 token 时 initialize 成功。
2. 返回 protocolVersion。
3. 返回 serverInfo。
```

---

### FR-BE-004：支持 tools/list

请求：

```json
{
  "jsonrpc": "2.0",
  "id": "tools-1",
  "method": "tools/list",
  "params": {}
}
```

响应：

```json
{
  "jsonrpc": "2.0",
  "id": "tools-1",
  "result": {
    "tools": [
      {
        "name": "coding.create_prd",
        "description": "Create PRD document",
        "inputSchema": {
          "type": "object",
          "properties": {
            "topic": {
              "type": "string"
            }
          },
          "required": ["topic"]
        }
      }
    ]
  }
}
```

验收：

```text
1. 返回 result.tools 数组。
2. 用户无授权时返回 tools: [] 或 MCP_FORBIDDEN。
3. 不允许返回 null。
```

---

### FR-BE-005：tools/list 权限过滤

最小权限链路：

```text
access_token
  -> user
  -> org_memberships
  -> organization
  -> user_skill_grants
  -> mcp_skill
  -> mcp_tool
```

规则：

```text
1. userId 必须来自 access token。
2. organizationId 来自当前用户组织上下文。
3. 只返回用户被授权 skill 下的 tools。
4. 禁用 skill 后 tools/list 不再返回对应 tools。
```

验收：

```text
1. 不同用户 tools/list 结果不同。
2. 无授权用户不返回任何 tools。
3. 权限变化后重新 tools/list 可生效。
```

---

### FR-BE-006：结构化错误码

| errorCode                  | JSON-RPC code | 说明            |
| -------------------------- | ------------: | ------------- |
| MCP_UNAUTHORIZED           |        -32001 | 未登录或 token 无效 |
| MCP_FORBIDDEN              |        -32003 | 无权限           |
| MCP_INITIALIZE_REQUIRED    |        -32010 | 未初始化          |
| MCP_TOOLS_LIST_FAILED      |        -32011 | tools/list 失败 |
| MCP_GATEWAY_REQUEST_FAILED |        -32012 | gateway 下游失败  |
| MCP_METHOD_NOT_FOUND       |        -32601 | 方法不存在         |
| MCP_INVALID_REQUEST        |        -32600 | 请求非法          |
| MCP_INTERNAL_ERROR         |        -32603 | 内部错误          |

要求：

```text
1. error.data.errorCode 必须存在。
2. 后端日志必须记录 errorCode、userId、orgId、method。
3. 禁止只返回 "MCP gateway request failed"。
```
