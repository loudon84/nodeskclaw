# v6.6 nodeskclaw MCP Skill Gateway 实施计划

## 前端表现变化

本次改动无前端表现变化。所有改动均为后端 API 层，服务于 Copilot Desktop 通过 MCP JSON-RPC 协议远程调用。

---

## 现状分析

项目已有完整的 MCP Skill Gateway 基础设施：

- **路由**：[`app/api/mcp_skill_gateway/router.py`](nodeskclaw-backend/app/api/mcp_skill_gateway/router.py) 已提供 `POST /mcp` 和 `GET /mcp/health`，挂载在 `/api/v1/` 下（即 `/api/v1/mcp` 和 `/api/v1/mcp/health`）
- **协议处理**：[`app/services/mcp_skill_gateway/handler.py`](nodeskclaw-backend/app/services/mcp_skill_gateway/handler.py) 已实现 `initialize`、`tools/list`、`tools/call` 三个 JSON-RPC method 的分发
- **认证**：[`app/services/mcp_skill_gateway/auth.py`](nodeskclaw-backend/app/services/mcp_skill_gateway/auth.py) 已实现 Bearer JWT 解析 -> User + Org 上下文
- **错误码**：[`app/services/mcp_skill_gateway/errors.py`](nodeskclaw-backend/app/services/mcp_skill_gateway/errors.py) 已定义 JSON-RPC 错误码体系
- **MCP Descriptor**：[`app/services/mcp_skill_gateway/constants.py`](nodeskclaw-backend/app/services/mcp_skill_gateway/constants.py) 的 `build_mcp_descriptor()` 已被 `/api/v1/system/info` 调用
- **工具映射**：[`app/services/hermes_skill/mcp_tool_mapper.py`](nodeskclaw-backend/app/services/hermes_skill/mcp_tool_mapper.py) 已实现基于 `HermesSkill` 数据库记录的工具列表和调用（面向 Hermes Skill Hub，创建异步 Task）

**关键差异**：PRD 要求的工具（`hermes.instances.list`、`hermes.instance.status`、`hermes.skills.list`）是**直接查询 external Docker Hermes 实例**的实时状态和技能列表，而现有 `McpToolMapper` 是基于**数据库中注册的 HermesSkill 记录**做工具暴露和异步任务创建。二者是两套独立的工具来源。

## 关键设计决策

### 1. 路由路径：`/api/v1/mcp` vs `/api/v1/hermes/mcp`

PRD 要求 `/api/v1/hermes/mcp`，现有实现为 `/api/v1/mcp`。

**方案：保持现有 `/api/v1/mcp` 不变，同时新增 `/api/v1/hermes/mcp` 作为别名指向同一 handler。** 理由：
- 现有 Copilot Desktop 代码已使用 `/api/v1/mcp`
- `system/info` 的 descriptor 中 `endpoint` 字段改为 `/api/v1/hermes/mcp`，新客户端用新路径
- 旧路径保持兼容

### 2. 工具注册方式：数据库 vs 硬编码

现有 `McpToolMapper` 从 `HermesSkill` 表查询工具。PRD 的三个工具是直接操作 Docker 的系统级工具，不依赖 Skill Hub 安装。

**方案：新增 `HermesDockerToolProvider` 类，硬编码三个系统工具定义，在 handler 中合并两种来源（Skill Hub + Docker Provider）。** 工具注册不入数据库，纯代码定义。

### 3. `instance_ref` 解析

PRD 定义 `instance_ref` 按 `instance_id -> container_name -> profile -> alias` 顺序解析。当前系统不存在此概念。

**方案：新增 `resolve_instance_ref()` 函数，查询 Instance 表，依次匹配 `id`、`advanced_config.external_container_name`、`advanced_config.profile`（或 slug/name）。**

### 4. 审计日志

PRD 要求 `mcp_call_logs` 表。项目已有 `McpGatewayAuditLog` 表结构较重（面向 Gateway Proxy 场景）。

**方案：新增轻量 `McpCallLog` Model，字段严格按 PRD B6 定义，用于 MCP Skill Gateway 调用审计。**

## 实施步骤

### Step 1: 新增 `McpCallLog` 审计 Model + Alembic 迁移

**文件**：
- 新增 `app/models/mcp_call_log.py`
- 更新 `app/models/__init__.py` 加入 import
- 生成 Alembic 迁移

**Model 定义**（按 PRD B6）：

```python
class McpCallLog(BaseModel):
    __tablename__ = "mcp_call_logs"

    org_id: Mapped[str] = mapped_column(String(36), nullable=False, index=True)
    user_id: Mapped[str] = mapped_column(String(36), nullable=False, index=True)
    tool_name: Mapped[str] = mapped_column(String(200), nullable=False, index=True)
    instance_id: Mapped[str | None] = mapped_column(String(36), nullable=True)
    status: Mapped[str] = mapped_column(String(20), nullable=False)
    duration_ms: Mapped[int | None] = mapped_column(Integer, nullable=True)
    input_summary: Mapped[dict | None] = mapped_column(JSONB, nullable=True)
    error_code: Mapped[str | None] = mapped_column(String(100), nullable=True)
    error_message: Mapped[str | None] = mapped_column(String(500), nullable=True)
```

继承 `BaseModel` 自动获得 `id`(UUID pk)、`created_at`、`updated_at`、`deleted_at`。

### Step 2: 新增 `HermesDockerToolProvider` — Hermes Docker 工具定义 + 执行

**文件**：新增 `app/services/mcp_skill_gateway/hermes_docker_tools.py`

**职责**：
- `list_tools()` — 返回三个工具的 MCP tool descriptor
- `call_tool(tool_name, arguments, org_id, user_id, db)` — 分发到对应 handler
- `resolve_instance_ref(ref, org_id, db)` — 按 PRD 顺序解析 instance_ref

**三个工具实现逻辑**（全部复用 `hermes_external` 服务层）：

| 工具 | 实现 |
|------|------|
| `hermes.instances.list` | 查询 Instance 表 where `binding_type=external_docker` + org，返回精简字段 |
| `hermes.instance.status` | `resolve_instance_ref()` -> `status_service.get_status(instance)` |
| `hermes.skills.list` | `resolve_instance_ref()` -> `skill_service.list_skills(instance)` -> 过滤返回精简字段 |

**`resolve_instance_ref` 实现**：

查询 Instance 表：
1. `id == ref` 且 binding_type 为 external_docker
2. `advanced_config->>'external_container_name' == ref`
3. `slug == ref` 或 `name == ref`

按此顺序，命中第一个即返回。

### Step 3: 增强 MCP Handler — 合并工具来源 + 写入审计

**文件**：修改 `app/services/mcp_skill_gateway/handler.py`

**改动**：
- `_handle_tools_list()` — 同时从 `McpToolMapper`（Skill Hub 工具）和 `HermesDockerToolProvider`（Docker 工具）获取工具列表，合并返回
- `_handle_tools_call()` — 判断 tool_name 前缀：
  - `hermes.instances.*` / `hermes.instance.*` / `hermes.skills.*` → 转发到 `HermesDockerToolProvider.call_tool()`
  - 其他 → 保持现有 `McpToolMapper.call_tool()` 逻辑
- 每次 `tools/call` 完成后写入 `McpCallLog` 审计记录（tool_name、duration_ms、status、input_summary 摘要、error_code/message）
- `input_summary` 只记录 key 和 value 长度，**不记录** Authorization、token、password 等敏感字段

### Step 4: 增强 MCP Health 端点

**文件**：修改 `app/api/mcp_skill_gateway/router.py`

**改动**：
- `GET /mcp/health` 返回格式从 `{"status":"ok"}` 扩展为：

```python
{
    "ok": True,
    "service": "nodeskclaw-mcp-skill-gateway",
    "status": "running",
    "tools": {"count": N}
}
```

- 需注入 db 依赖以统计工具数量（Skill Hub 工具数 + Docker 固定 3 个）

### Step 5: 新增 `/api/v1/hermes/mcp` 别名路由

**文件**：修改 `app/api/mcp_skill_gateway/router.py` 和 `app/api/router.py`

**改动**：
- 在 `router.py` 新增 `POST /hermes/mcp` 端点，调用同一 `dispatch()` 函数
- `router.py` 中注册时 tag 标注为 `MCP Skill Gateway`
- 原有 `/mcp` 路由保留兼容

### Step 6: 更新 MCP Descriptor

**文件**：修改 `app/services/mcp_skill_gateway/constants.py`

**改动**：
- `MCP_ENDPOINT` 改为 `/api/v1/hermes/mcp`（PRD 对齐）
- 新增 `MCP_ENDPOINT_LEGACY = "/api/v1/mcp"`（向后兼容）
- `build_mcp_descriptor()` 使用新 endpoint

### Step 7: 工具权限分级（read/write/admin）

**文件**：在 `app/services/mcp_skill_gateway/hermes_docker_tools.py` 中定义

**设计**：

```python
TOOL_PERMISSION_LEVEL = {
    "hermes.instances.list": "read",
    "hermes.instance.status": "read",
    "hermes.skills.list": "read",
    # 第二阶段
    "hermes.skills.install_builtin": "write",
    "hermes.skills.install_zip": "write",
    "hermes.skills.install_git": "write",
    "hermes.skills.uninstall": "write",
    "hermes.instance.restart": "admin",
}
```

本阶段 `call_tool()` 只允许 `read` 级别工具，`write`/`admin` 返回 `MCP_FORBIDDEN`。权限检查依赖现有 `instance_member_service.check_instance_access()` — read 只需 `InstanceRole.viewer`。

### Step 8: GeneHub 工具预留（桩方法）

**文件**：在 `hermes_docker_tools.py` 中预留

**改动**：
- 在 `list_tools()` 中**不返回** GeneHub 工具（本阶段不暴露）
- 但在代码中定义好四个工具签名（`genehub.skills.search` 等），注释标注为 Phase 3
- `call_tool()` 中遇到 `genehub.*` 前缀返回 `"not_implemented"` 错误

### Step 9: 审计日志 Service

**文件**：新增 `app/services/mcp_skill_gateway/audit_service.py`

**职责**：
- `log_mcp_call()` — 将 `McpCallLog` 写入数据库
- 自动脱敏 `input_summary`：移除 `authorization`、`token`、`password`、`webui_password` 等字段
- 支持异步写入（`asyncio.create_task` 或在 handler 同步写）
