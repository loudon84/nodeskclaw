# v4.0 Hermes MCP Skill Gateway 运行时闭环实施计划

## 前端表现变化

### 1. Portal - 新增 Hermes Tasks 页面

**总结**: Portal 新增任务列表页，用户可查看 Hermes Task 状态、事件流和产物

**元素级变化**:
- 导航菜单: **新增** "Hermes Tasks" 入口
- Task 列表表格: **新增**，展示 task_no / tool_name / agent_id / status / created_at
- Status 筛选下拉: **新增**，支持按状态过滤（queued/running/completed/failed 等）
- Task 详情抽屉/面板: **新增**，展示完整 task 信息 + 事件流 + Artifact 列表
- Cancel 按钮: **新增**，queued/accepted/running 状态下可用
- Retry 按钮: **新增**，failed/timeout 状态下可用
- SSE 事件流面板: **新增**，实时展示 task 事件

**改动后**:
```
/portal/hermes/tasks
+-- Hermes Tasks -----------------------------------------+
| [Status: All v]                                          |
| +------+-------------------+----------+--------+-------+ |
| | TASK | Tool              | Agent    | Status | Time  | |
| +------+-------------------+----------+--------+-------+ |
| | 0001 | writer_article... | writer.. | queued | 12:00 | |
| | 0002 | coder_review...   | coder..  | done   | 11:30 | |
| +------+-------------------+----------+--------+-------+ |
|                                                          |
| [<< Task Detail >>]                                      |
| Status: running                                          |
| Events:                                                  |
|   task.created -> task.queued -> task.accepted            |
| Artifacts: (0)                                           |
| [Cancel]  [Retry]                                        |
+----------------------------------------------------------+
```

### 2. Portal - 新增 Hermes Artifacts 页面

**总结**: Portal 新增产物列表页，用户可浏览、预览、下载 Artifact

**元素级变化**:
- 导航菜单: **新增** "Hermes Artifacts" 入口
- Artifact 列表表格: **新增**，展示 file_name / content_type / size / download_count
- Preview 按钮: **新增**，text/* / application/json / image/* 类型可用
- Download 按钮: **新增**
- 筛选区域: **新增**，支持 task_id / skill_id / content_type 过滤

### 3. 后端 API 变化（无直接 UI 但影响前端对接）

**总结**: MCP tools/call 返回结构从 content-only 改为 content + structuredContent

## 实施步骤

### Epic 1: MCP tools/call 响应与 TaskService 收敛

**涉及文件**:
- 修改: [nodeskclaw-backend/app/services/hermes_skill/mcp_tool_mapper.py](nodeskclaw-backend/app/services/hermes_skill/mcp_tool_mapper.py)
- 修改: [nodeskclaw-backend/app/services/hermes_skill/task_service.py](nodeskclaw-backend/app/services/hermes_skill/task_service.py)
- 修改: [nodeskclaw-backend/app/services/mcp_skill_gateway/handler.py](nodeskclaw-backend/app/services/mcp_skill_gateway/handler.py)

**改动**:

1. **TaskService.create_task** 补齐:
   - 在 task.created 事件后补写 task.queued 事件（event_seq=1）
   - 设置 `task.timeout_seconds = settings.HERMES_TASK_DEFAULT_TIMEOUT_SECONDS`
   - 不在内部 commit，由调用层管理事务

2. **McpToolMapper.call_tool** 改为调用 TaskService:
   - 删除手写 HermesTask 创建 + HermesTaskEvent 创建 + commit 逻辑
   - 改为 `task = await TaskService(self.db).create_task(...)`
   - 保留 tool_name 查找、installation 查找、权限校验、input_schema 校验
   - 返回 dict 含 `tool_name / agent_id / status / task_id / task_no / event_url / artifact_url`

3. **handler.py `_handle_tools_call`** 对 Hermes Skill tool 返回 structuredContent:
   - 识别 mapper.call_tool 返回的 result 含 `task_id` 时，包装为:
     ```python
     content = [{"type": "text", "text": "任务已创建"}]
     structuredContent = result
     ```
   - 新增辅助函数 `_hermes_skill_tool_call_success(jsonrpc_id, result)` 区分于通用的 `_tool_call_success`

### Epic 2: Task API 补齐

**涉及文件**:
- 修改: [nodeskclaw-backend/app/api/hermes_skill/tasks_router.py](nodeskclaw-backend/app/api/hermes_skill/tasks_router.py)

**改动**:

1. 新增 `GET /tasks` 列表接口:
   - 查询参数: page / page_size / status / skill_id / tool_name / agent_id / profile_id / workspace_id / user_id
   - 调用 `TaskService.list_tasks`（已实现）
   - 权限校验: `hermes_task:view`
   - 返回分页结构: `{ items, total, page, page_size }`

### Epic 3: HermesTaskWorker 加固

**涉及文件**:
- 修改: [nodeskclaw-backend/app/services/hermes_skill/hermes_task_worker.py](nodeskclaw-backend/app/services/hermes_skill/hermes_task_worker.py)

**改动**:

1. `_fetch_and_lock` 中 accepted 事件去重:
   - 写 accepted 事件前查询是否已存在 `TASK_ACCEPTED` 事件，存在则跳过

2. `_execute_task` 中 running 事件去重:
   - 写 running/started 事件前查询是否已存在

3. completed 后 artifact scan 失败隔离:
   - 将 artifact scan 包在独立 try/except 中，scan 失败写 `artifact.scan.failed` 事件但不影响 task 的 completed 状态
   - 确保 `task.completed` 事件和 flush 已执行后再 scan

4. failed/timeout 必须写 error_code/error_message:
   - 确认所有 FAILED 路径都设置了 error_code 和 error_message（当前已有，review 补漏）

5. worker lock 释放稳定:
   - 确保 `_execute_task` 结束时（包括异常路径）始终清理 `worker_id` / `locked_at`

### Epic 4: HermesAgentAdapter 加固

**涉及文件**:
- 修改: [nodeskclaw-backend/app/services/hermes_skill/hermes_agent_adapter.py](nodeskclaw-backend/app/services/hermes_skill/hermes_agent_adapter.py)

**改动**:

1. `_get_base_url` 补 `endpoint_url` fallback:
   - 在 `gateway_url` 和 `ingress_domain` 之间加入 `instance.endpoint_url` 检查

2. `_get_instance` 增加 org_id 校验:
   - 方法签名加 `org_id` 参数，查询条件加 `Instance.org_id == org_id`
   - 所有调用点传入 `task.org_id`

3. `get_run_status` 兼容多种响应格式:
   - 依次尝试 `run_data.get("status")` / `run_data.get("state")` / `run_data.get("data", {}).get("status")`

4. `cancel_run` 兼容 DELETE 和 POST:
   - 先尝试 DELETE，若 4xx 则回退 POST `/v1/runs/{id}/cancel`

### Epic 5: ArtifactService 路径闭环

### Epic 6: SkillInstaller 安装到真实 Profile

**涉及文件**:

**改动**:

1. `_build_target_path` 中 hermes_agent 类型缺 profile_root_path 时报错:
