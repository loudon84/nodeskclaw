## 架构决策（已与用户确认）

- 数据模型：**复用扩展现有 `HermesArtifact`**，不新建 `hermes_task_artifacts`。物化 Markdown 走中心对象存储（`storage_service.upload_raw`），复用现有 preview/download/权限链路。
- 端点冲突：现有 `GET /tasks/{task_id}/artifacts` 保持 `data` 数组兼容，**追加 `server_artifacts` 字段**（物化产物子集）。
- 前端范围：后端 + Router Skill 模板 + portal 任务详情 server_artifacts 卡片 + **KB 入库审核管理页**。

## 现状关键事实（已核实）

- `McpToolMapper.call_tool()` 在 [mcp_tool_mapper.py](nodeskclaw-backend/app/services/hermes_skill/mcp_tool_mapper.py)，构建 `routing_metadata` 并 `TaskService.create_task`，返回 dict → handler `_hermes_skill_tool_call_success` 包成 `structuredContent`（[handler.py](nodeskclaw-backend/app/services/mcp_skill_gateway/handler.py)）。tools/call 仅返回 queued 态，completed 态产物通过 `result_url` 轮询。
- 任务全文在 worker：[hermes_task_worker.py](nodeskclaw-backend/app/services/hermes_skill/hermes_task_worker.py) `mark_completed(task, result_summary=content_text[:500])`。`result_summary` 是 `String(500)` 截断，**物化必须用 worker 的全文 `content_text`，不能读 DB**。
- 已有 `HermesArtifact`（[hermes_artifact.py](nodeskclaw-backend/app/models/hermes_skill/hermes_artifact.py)）+ `ArtifactService` 的 preview/download（按 `file_path` 读本地 FS）+ 权限链 `PermissionChecker.can_view/download_artifact`。
- 已有 `storage_service.upload_raw/download_raw/delete_raw`（S3+本地双后端，[storage_service.py](nodeskclaw-backend/app/services/storage_service.py)），直接用作 Artifact Store。
- `HermesSkill` 有 `extra_metadata` 但无 `output_policy` 列。`result_url`/`artifact_url` 由 `task_result_router`/`tasks_router` 提供。

## 前端表现变化

### 1. Portal 任务详情 — 新增「中心产物库」卡片

**总结**: 任务详情从「只展示 result_summary + discovery 产物」变为「额外展示 Gateway 物化到中心产物库的 server_artifacts，含预览/下载/建议导入路径/知识库状态」。

**元素级变化**:
- 「中心产物库」卡片: **新增**，位于任务详情结果区
- 每条产物行: **新增** 文件名 + 类型 badge（markdown/json/txt）、预览按钮、下载按钮、`suggested_workspace_path` 只读文本 + 复制按钮、`kb_status` badge（待审核/已入库/已拒绝/未入库）
- 预览交互: **新增** 点击预览弹出 Markdown/文本预览（复用现有 preview API，超 200KB 显示「已截断」）
- 空状态: **新增** 无 server_artifacts 时显示「本任务无中心产物」
- 提示文案: **新增** 「报告已保存到 nodeskclaw 中心产物库，可下载或按建议路径导入当前 workspace」

**改动后**:
```

│                                             │
│ ┌─ 中心产物库 ─────────────────────────┐   │

```

### 2. KB 入库审核管理页（新增）

**总结**: 新增 `/hermes/kb-ingestion` 管理页，管理员/operator 审核中心产物入库通用知识库（默认 pending_review，不自动入库）。

**元素级变化**:
- 侧边导航入口「知识库入库」: **新增**
- 入库任务列表: **新增**（卡片/列表，非 table），每条含 产物名、目标 knowledge_base、tags、状态 badge、来源 tool_name、创建时间
- 状态筛选器: **新增**（pending_review / approved / indexed / rejected）
- 「通过」按钮: **新增**，点击 → 状态流转 approved→indexing→indexed
- 「拒绝」按钮 + 理由弹窗: **新增**，填写 review_comment
- 空状态/加载态/成功失败 toast: **新增**

## 后端实施

### Task 1：数据模型与迁移

- 扩展 [hermes_task.py](nodeskclaw-backend/app/models/hermes_skill/hermes_task.py)：新增 `output_policy JSONB`、`server_artifacts JSONB`、`artifact_status VARCHAR(32) default 'none'`、`kb_status VARCHAR(32) default 'none'`。
- 扩展 [skill.py](nodeskclaw-backend/app/models/hermes_skill/skill.py)：新增 `output_policy JSONB NULL`。
- 新建 `HermesArtifactKbIngestionJob` 模型（`hermes_artifact_kb_ingestion_jobs`，PRD §9.3，继承 `BaseModel` 软删除），状态机字段 + tags JSONB + reviewed_by/at/comment + indexed_at/index_error + `sha256`（去重）。
- 迁移：`uv run alembic revision --autogenerate -m "v5.6 mcp artifact bridge"`（Review，确认 partial unique / 默认值）。

### Task 2：OutputPolicyService
- 新建 [output_policy_service.py](nodeskclaw-backend/app/services/mcp_skill_gateway/output_policy_service.py)
- `resolve(skill, installation, tool_name)`：优先级 `installation.routing_metadata.output_policy > skill.output_policy > DEFAULT_POLICIES`。
- `DEFAULT_POLICIES`：PRD §7.2 字典，key 用 tool_name 去掉 `hermes_xxx__` 前缀的短名（如 `semiconductor-marketing-copy`）。固定 `artifact_mode=pull_only`。
- 缺省兜底策略（未命中默认表）：markdown + `drafts/misc` + `kb_ingest.enabled=false`。

### Task 3：ArtifactMaterializer
- 新建 [artifact_materializer.py](nodeskclaw-backend/app/services/mcp_skill_gateway/artifact_materializer.py)
- `materialize(task, full_result_text, output_policy)` → `ArtifactContent(filename, content_bytes, mime_type, format, suggested_workspace_dir, suggested_workspace_path)`。
- markdown：YAML frontmatter（source/task_id/tool_name/artifact_id/created_at/kb_status，PRD §8.3）+ 正文；json/txt 分支。
- `render_filename(template, task)`：替换 `{topic}/{company}/{date}` 等（从 `task.arguments` 取 topic/company，date=完成时间），非法字符清洗。
- `build_suggested_workspace_path(dir, filename)` → `workspace/{dir}/{filename}`。
- **安全**：frontmatter/正文不得写入 token/Authorization。

### Task 4：ArtifactStoreService（薄封装复用 storage_service）
- 新建 [artifact_store_service.py](nodeskclaw-backend/app/services/mcp_skill_gateway/artifact_store_service.py)
- `store(org_id, task_id, artifact_id, content)`：object_key=`orgs/{org_id}/tasks/{task_id}/artifacts/{artifact_id}/{filename}`（PRD §8.2），调用 `storage_service.upload_raw`，返回 key/size/sha256。
- `read(object_key)` → `storage_service.download_raw`。

### Task 5：ServerArtifactService（编排）
- 新建 [server_artifact_service.py](nodeskclaw-backend/app/services/mcp_skill_gateway/server_artifact_service.py)
- `create_from_task_result(task, full_result_text, output_policy)`：materialize → store → 创建 `HermesArtifact`（`source=materialized`, `storage_type=object_store`, `object_key`, `kb_status`, `suggested_workspace_*`, `artifact_type/format`, `permission_scope` 复用现有 org/workspace 逻辑）→ 触发 KbIngestionService → 返回 `server_artifacts`（PRD §5.1 形态：artifact_id/name/type/mime_type/stored/store/download_url/preview_url/suggested_workspace_path/workspace_saved/kb_status）。
- download_url/preview_url 复用现有扁平端点 `/api/v1/hermes/artifacts/{id}/preview|download`。
- `store_to_gateway=false` 时跳过。审计事件 `mcp_artifact.materialize.*` / `store.completed`。

### Task 6：KbIngestionService

## 风险与注意

- `result_summary` 500 字限制：物化全文不入 summary，server_artifacts 单独存。
- object_store 产物的 preview/download 必须绕过现有 FS path_guard（按 storage_type 分支），否则会因 path_guard 校验失败。
- KB「indexing→indexed」真实索引执行器 v5.6 仅为 hook（PRD §3 不强制自动入库、§15.4 仅 sha256 去重）。
- 安全：artifact 内容与 frontmatter 绝不写 token/Authorization。
